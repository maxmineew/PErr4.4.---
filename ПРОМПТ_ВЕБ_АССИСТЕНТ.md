# Промпт: Создание веб-сайта с ИИ-ассистентом для сбора заявок

**Роль:** Fullstack-разработчик + AI-инженер

## Задача

Реализовать чат-виджет для лендинга, который принимает заявки от пользователей, ведёт диалог с помощью ИИ и сохраняет данные в Яндекс Таблицу. Взаимодействие: frontend → backend (Flask), логика — пошаговый сбор информации.

---

## Логика ассистента

### Собираемые данные (пошагово)

1. **Имя** — как к пользователю обращаться
2. **Контакт** — телефон или email
3. **Описание запроса** — суть обращения

### Поведение

- Вежливый, нейтральный тон
- Задаёт уточняющие вопросы по запросу
- **Не уходит в продажи** — фокус на сборе заявки, без навязывания товаров/услуг
- Не придумывает данные за пользователя
- Если пользователь уходит от темы — мягко возвращает к сбору заявки
- Один вопрос за один ход

---

## Интерфейс чат-виджета

- **Стиль:** минималистичный, тёмный нейтральный дизайн
- **Размещение:** справа внизу экрана (фиксированная кнопка/окно)
- **Функции:**
  - Автопрокрутка к последнему сообщению
  - Индикатор «Ассистент печатает» при ожидании ответа ИИ
- **Тип сайта:** лендинг (одностраничник)
- **Доступность:** чат всегда доступен, без скрытия за страницами

---

## Архитектура

### Backend (Flask)

- Один Flask-приложение
- Минимум зависимостей: `flask`, `requests` (YandexGPT), `gspread` или `yadisk` + `openpyxl` (таблицы), `python-dotenv`
- Endpoints:
  - `GET /` — отдача лендинга + встроенный чат
  - `POST /api/chat` — приём сообщения пользователя, вызов ИИ, возврат ответа
  - `POST /api/submit` — сохранение заявки в таблицу (после сбора данных)

### Frontend

- Один HTML-файл с инлайн CSS/JS (или один `static/` каталог)
- Чат-виджет — ванильный JS, без фреймворков
- Запросы к backend через `fetch`

### Хранение данных

- **Яндекс Таблицы:** отдельный лист **«Заявки с сайта»**
- Структура данных отличается от Telegram-бота (другой сценарий)
- Колонки: `Имя`, `Контакт`, `Описание запроса`, `Дата`, `Источник (сайт)`
- Использовать API Яндекс Таблиц или Яндекс Диск + Excel — см. реализацию в Telegram-боте

## Технические ограничения

1. **Только необходимые файлы** — никаких `run.bat`, `run.sh`, лишних скриптов установки
2. **Готово к деплою** — `requirements.txt`, инструкция в `README.md`
3. **Запуск:** `flask run` или `gunicorn` для продакшена
4. **Структура простая и читаемая:**
   ```
   app/
   ├── __init__.py      # Flask app
   ├── routes.py        # /api/chat, /api/submit
   ├── ai_logic.py      # YandexGPT, сценарий сбора
   ├── sheets.py        # Запись в таблицу
   ├── config.py        # Переменные окружения
   static/
   ├── index.html       # Лендинг + чат
   templates/           # (если используешь Jinja)
   requirements.txt
   README.md
   ```
5. **Без лишнего кода** — только функциональность заявок, без админки, авторизации и т.п.

## Переменные окружения

- `FLASK_SECRET_KEY` — секрет приложения
- `YANDEX_API_KEY`, `YANDEX_FOLDER_ID` — YandexGPT
- `YANDEX_DISK_TOKEN`, `YANDEX_DISK_FILE_PATH` — таблица
- Лист в таблице: **«Заявки с сайта»**

## Итог

Создать Flask-приложение с минималистичным тёмным чат-виджетом (справа внизу), ИИ-ассистентом для сбора имени, контакта и описания запроса, сохранением в отдельный лист «Заявки с сайта». Код читаемый, структура простая, проект готов к деплою без лишних скриптов и файлов.
